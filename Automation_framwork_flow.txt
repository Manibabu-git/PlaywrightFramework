                                                                                                                                                                                                                                                                                                                             Details of every package/directory created within the framework are as follows:
 
Building Blocks for any script:
Pages: Comprises of locators and their associated action methods. Locators are first declared as readonly variables and later defined within class constructor.
testConfig: Comprises of all different permutations for VM creation. Specific to IaaS.
TestData: Comprises of all test data specific to NaaS.
tests: Comprises of all the tests. Separated as per PODs and catalog items.
Utilities: It is further sub-divided into: 
    apihelpers: Code related to Api calls. Used in qmetry update.
    dbhelpers: Code to connect to MongoDB. Call this function to connect and query data from MongoDB.
    enums: To save all static UI specific data like catalog list, dropdown values, etc.  
    remotehelpers: Code related to validate json files being created for VM intake form, myrequest page data, my resources page data and data coming from remote VM.
    uihelpers: All UI specific common utility methods like screenshot capture, waits, json data creation, etc.
    vmValidator: Code to login to remote VM and perform actions. Python utility 
 
Supporting Packages:
fixtures: Contains before and after codes such as session creation, initializing chrome instance and playwright configs.
globalConfig: Code to cleanup reports folder, storing captured videos to videos folder, global constants.1.
 
Output Folders:
allure-report: Will store downloadable report that can be shared across.
allure-results: Will store execution reports. Above allure-report will be created using files from here.
test-results: Will store individual test script report - all captured screenshots, videos and trace specific to test script. Use this report for debugging purposes.
TestResponse: Will store all important nebula related responses such as request Ids, hostnames, ipAddresses, installed tool records.
videos: Use this to store videos, if uploading it to qmetry.
VM_Validation_Results: Will store all comparison file between VM intake form vs MyRequest Page vs MyResources Page vs Remote VM.
 
Configuration files:
.env: To save all environments data such as test urls, user and approval credentials, mongo connection strings, etc.
playwright.config.ts: To save all playwright configs. Set browsers to run from here, parallel counts, turn video/screenshot capturing ON/OFF from here.
